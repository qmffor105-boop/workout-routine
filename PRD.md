## 1. 제품 개요

### 1.1 한 줄 설명
집에서 맨몸·간단한 도구로 수행할 수 있는 **4주 프로그레시브 홈트레이닝 루틴**을, 목표(체력향상/벌크업/다이어트)에 맞춰 요일별로 제공하는 **싱글 파일 웹앱**.

### 1.2 문제 정의
- 홈트를 시작하려는 초·중급자는 "오늘 뭘 할지"부터 막힌다.
- 유튜브 루틴은 산발적이고 연속성(주차별 강도 증가)이 없다.
- 헬스장 PT는 비용 부담이 크고, 앱은 가입·결제 장벽이 있다.

### 1.3 타깃 사용자
- **1차**: 20~40대 홈트 입문/재시작자, 주 3~5회 운동 의지는 있으나 프로그램이 없는 사람
- **2차**: 출장·재택 등으로 헬스장 이용이 불규칙한 중급자

### 1.4 성공 지표 (초기 MVP)
- 1회 방문 시 평균 세션 시간 ≥ 3분
- 목표(goal) 또는 주차(week) 변경 발생률 ≥ 40%
- 운동 체크 인터랙션 발생률 ≥ 25%

## 2. 범위

### 2.1 In-Scope (MVP)
- 3가지 목표 루틴: **체력향상 / 벌크업 / 다이어트**
- 4주 프로그레시브 프로그램 (1~3주 점증, 4주 디로드)
- 요일별 운동 카드 (카디오/근력/코어/플라이오/스트레칭 뱃지)
- 운동 완료 체크 및 진행률 표시
- 인터벌 타이머 (프리셋 포함)
- 루틴 셔플
- 주차별 코칭 팁
- 모바일 반응형 레이아웃
- GitHub Pages 정적 호스팅

### 2.2 Out-of-Scope (현재)
- 사용자 계정 / 로그인
- 서버 데이터 동기화
- 운동 영상·GIF 임베드
- 식단 관리, 체중 기록
- 푸시 알림 / PWA 설치

## 3. 사용자 스토리

1. **목표 선택** — "다이어트를 하려고 한다. HIIT 위주 루틴을 보고 싶다." → 상단 목표 버튼 클릭 시 루틴 전체가 교체된다.
2. **요일별 루틴 확인** — "오늘은 월요일이다. 오늘의 운동이 뭔지 즉시 보고 싶다." → 탭에서 해당 요일 클릭 → 카드 목록 렌더링.
3. **운동 체크** — "한 세트 끝냈다. 표시해두고 싶다." → 카드 클릭 시 완료 상태로 전환, 진행률 바 업데이트.
4. **휴식 타이머** — "세트 간 60초 쉬어야 한다." → 타이머 프리셋 클릭 → 카운트다운.
5. **지루함 방지** — "같은 루틴을 반복하니 지겹다." → 셔플 버튼 → 같은 목표 내 운동 순서 재배치.
6. **주차 이동** — "다음 주는 어떻게 될지 미리 보고 싶다." → 주차 선택 버튼.

## 4. 기능 요구사항

### 4.1 목표 선택기 (Goal Selector)
- 표시 목표: 체력향상 / 벌크업 / 다이어트
- 목표별 강조 컬러(시안 / 오렌지 / 핑크)로 현재 선택 상태 표시
- 목표 전환 시 주차 및 요일 루틴이 동기화되어 재렌더링

### 4.2 주차 선택기 (Week Selector)
- 1주 ~ 4주 버튼
- 4주차는 **디로드** 주간임을 팁으로 명시

### 4.3 요일 탭 & 운동 카드
- 요일 탭: 월~일 + 휴식일 시각 구분
- 카드 필드: 운동명, 카테고리 뱃지, 세트/렙/시간, 보조 설명
- 완료 체크: 카드 클릭 시 상태 토글, 텍스트 취소선 + 진행률 바 반영

### 4.4 인터벌 타이머
- 디스플레이: MM:SS, 시작/정지/리셋
- 프리셋: 30s / 45s / 60s / 90s / 2m (이상)
- 0 도달 시 시각적 알림

### 4.5 셔플
- 현재 선택된 목표·주차·요일 내에서 운동 순서 랜덤화
- 체크 상태는 운동 ID 기준으로 유지

### 4.6 코칭 팁
- 주차 × 목표마다 1개 이상 제공
- 수면, 단백질, 강도 조절 등 실용 조언

## 5. 비기능 요구사항

- **배포**: GitHub Pages (main 브랜치 / 루트)
- **성능**: 초기 로드 < 2초 (3G 기준), 상호작용 < 100ms
- **접근성**: 버튼 최소 터치 타겟 44px, 색상 대비 WCAG AA 이상 지향
- **호환성**: 최신 Chrome/Safari/Edge/Firefox, iOS 15+ / Android 10+
- **보안**: CSP 헤더로 스크립트 소스 제한, 외부 네트워크 의존 최소화
- **프라이버시**: 사용자 데이터 서버 전송 없음 (클라이언트 전용)
